## 技术栈（1.5分钟）

### 硬件

- **M5Capsule** — 比 AirPods 还小
- 磁吸设计，可以贴在显示器上
- 250mAh 电池，续航 40 小时

### 软件

- **固件**：PlatformIO + Arduino
- **通信**：USB Serial 配置同步（LED 亮度、提示音音量、番茄时长）
- **网页端**：https://qqshi13.github.io/tools-suite/web-sync.html
- **离线运行**：配置完成后无需连接电脑

### AI 怎么帮我

"整个开发过程中，OpenClaw 的 Agent（Nova）帮我：

- 设计低功耗策略（CPU 降到 80MHz）
- 调试 USB 通信协议
- 优化 LED 亮度算法

不是写代码工具，是**结对编程伙伴**。"

## OpenClaw 到底是什么？（1.5分钟）

**不只是 API 调用工具**

传统方式：

```
你 → 写 Prompt → 得到回答 → 自己执行
```

OpenClaw 方式：

```
你 ←→ Agent（有记忆、有工具、有个性）
     ↓
   自动执行（查资料、改代码、发消息）
```

### 关键区别

| 传统 Chat  | OpenClaw Agent     |
| -------- | ------------------ |
| 每次对话从零开始 | 记得你的项目、偏好、历史       |
| 只会说      | 会读文件、运行命令、操作浏览器    |
| 通用回答     | 有个性文件（SOUL.md），有态度 |

### 我的使用经验

**1. 模型选择：OpenClaw vs 专用 Coding Agents**

| 场景 | 推荐工具 | 为什么 |
|------|---------|--------|
| 复杂问题/综合任务/日常开发 | **OpenClaw** | 有完整记忆系统，能操作浏览器、发消息、管理文件，上下文无限 |
| 大型代码库重构 | **Claude Code** | 代码理解强，适合复杂重构 |
| 快速代码生成 | **Kimi Code / Codex** | 速度快，但上下文有限 |
| 探索性项目 | **OpenClaw** | 从想法到部署全流程陪伴 |

**关键区别：**

Coding Agents（Claude Code/Kimi/Codex）：
- 专注写代码，上下文有限（Kimi 只有几十轮）
- 每次启动从零开始，不记得你的项目历史
- 不会主动帮你查资料、发消息、管理任务

OpenClaw：
- **记忆持久化** —— 每天对话都记住，像长期合作伙伴
- **工具丰富** —— 浏览器、文件系统、消息、定时任务
- **有个性** —— SOUL.md 定义性格，不是冷冰冰的工具
- **无限上下文** —— 不会聊着聊着就忘了之前的事

**我的组合用法：**
- 日常所有任务 → OpenClaw（Nova）
- 需要大规模重构 → 临时开 Claude Code
- 快速写片段 → Kimi Code
- 结果都回到 OpenClaw 统一管理

**2. 建立上下文**

- 用 memory 文件持久化信息
- 创建技能复用工作流
- 用 cron 定时任务自动化

**3. 把它当人**

- 写 SOUL.md 定义性格
- 写 USER.md 介绍自己
- 长期对话建立默契

## Q&A 准备

**可能的问题：**

Q: 为什么不用现成的番茄钟？
A: 想要和数字工作流打通，还要物理反馈。现有产品要么太封闭，太贵。

Q: 制作成本多少？
A: M5Capsule 大概145元。

Q: OpenClaw 和直接调用 API 有什么区别？
A: 记忆持久化 + 工具调用 + 多轮对话上下文。不是一问一答，是长期合作关系。

Q: 代码开源吗？
A: 是的，GPL-3.0，在 GitHub/QQSHI13/M5Timer。
