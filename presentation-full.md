## 技术栈（1.5分钟）

### 硬件
- **M5Capsule** — 比 AirPods 还小
- 磁吸设计，可以贴在显示器上
- 250mAh 电池，续航 40 小时

### 软件
- **固件**：PlatformIO + Arduino
- **通信**：USB Serial 配置同步（LED 亮度、提示音音量、番茄时长）
- **网页端**：https://qqshi13.github.io/tools-suite/web-sync.html
- **离线运行**：配置完成后无需连接电脑

### AI 怎么帮我
"整个开发过程中，OpenClaw 的 Agent（Nova）帮我：
- 设计低功耗策略（CPU 降到 80MHz）
- 调试 USB 通信协议
- 优化 LED 亮度算法

不是写代码工具，是**结对编程伙伴**。"

---

## OpenClaw 到底是什么？（1.5分钟）

**不只是 API 调用工具**

传统方式：
```
你 → 写 Prompt → 得到回答 → 自己执行
```

OpenClaw 方式：
```
你 ←→ Agent（有记忆、有工具、有个性）
     ↓
   自动执行（查资料、改代码、发消息）
```

### 关键区别

| 传统 Chat | OpenClaw Agent |
|-----------|----------------|
| 每次对话从零开始 | 记得你的项目、偏好、历史 |
| 只会说 | 会读文件、运行命令、操作浏览器 |
| 通用回答 | 有个性文件（SOUL.md），有态度 |

### 我的使用经验

**1. 模型选择**
- 复杂问题 → Kimi/K2.5（推理强）
- 快速任务 → Doubao（速度快）
- 代码生成 → Claude Code

**2. 建立上下文**
- 用 memory 文件持久化信息
- 创建技能复用工作流
- 用 cron 定时任务自动化

**3. 把它当人**
- 写 SOUL.md 定义性格
- 写 USER.md 介绍自己
- 长期对话建立默契

## Q&A 准备

**可能的问题：**

Q: 为什么不用现成的番茄钟？
A: 想要和数字工作流打通，还要物理反馈。现有产品要么太封闭，太贵。

Q: 制作成本多少？
A: M5Capsule 大概 60 元，加上磁吸配件不到 70。

Q: OpenClaw 和直接调用 API 有什么区别？
A: 记忆持久化 + 工具调用 + 多轮对话上下文。不是一问一答，是长期合作关系。

Q: 代码开源吗？
A: 是的，GPL-3.0，在 GitHub/QQSHI13/M5Timer。
