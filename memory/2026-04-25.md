

## M5Timer Buzzer Crisis (2026-04-26 09:00)
- User flashed firmware at wrong offset (0x0000 instead of 0x10000) — bricked device
- After recovery, LED and sound were at max — NVS settings corrupted from bad flash
- Root cause: Commit `6b83db9` changed buzzer to `ledcAttach()` (ESP32 Arduino 3.x API)
- User's build environment uses ESP32 Arduino 2.x API: `ledcSetup` + `ledcAttachPin`
- Reverted to stable version, fixed buzzer API mismatch
- Built and flashed successfully — device recovered
- Committed: `d54d232` (M5Timer) + `056ae87` (workspace)
- Recovery files on Desktop: `M5Timer-firmware.bin`, `M5Timer-bootloader.bin`, `M5Timer-partitions.bin`
